# run-sync — ETL Orkestratörü

**Özet:** MSSQL kaynak veritabanından PostgreSQL hedef veritabanına veri transferini yöneten ana orkestratör. Lookup sync'i önce, sales sync'i sonra çalıştırır; eşzamanlı sync'i kilit mekanizmasıyla engeller.
**Kütüphaneler:** Python `logging`, `sync_lock`, `sync_lookup`, `sync_sales`
**Bağlantılar:** [[entities/sync-worker]], [[entities/sync-lookup]], [[concepts/etl-pipeline]], [[decisions/karar-sync-worker-ayri]]

## Sync Sırası (Zorunlu)

```
run_full_sync()
  1. is_sync_running() → varsa abort
  2. run_lookup_sync()  → Marka, Ürün, Kategori, Mağaza, Kampanya, Müşteri
     → başarısız olursa sales atlanır
  3. run_sales_sync(full_sync=False)  → aylık artımlı satış
```

Lookup önce gelmek zorundadır — satış verileri lookup'taki FK'lara referans verir.

## CLI Arayüzü
```
python run_sync.py              # Tam sync
python run_sync.py --lookup     # Sadece lookup
python run_sync.py --sales      # Artımlı satış
python run_sync.py --sales-full # Tam satış geçmişi
python run_sync.py --status     # Tablo istatistikleri
```
